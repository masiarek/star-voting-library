# method_comparisons — same ballots, different methods

The library's crown jewels: teaching sets where the *contrast between methods*
is the lesson. Each set keeps its matched files together — splitting them into
per-method folders would destroy the pedagogy.

| Set | The lesson |
|---|---|
| [`black_curtain/`](black_curtain/README_black_curtain.md) | four elections, identical first-choice "landslides" — Approval, Score, STAR, and RCV-IRV reveal (or hide) four very different electorates |
| [`center_squeeze/`](center_squeeze/README_center_squeeze.md) | the same 1-D electorate: RCV-IRV eliminates the consensus center; STAR elects it |
| [`monotonicity/`](monotonicity/README_monotonicity.md) | before/after pairs where MORE support makes a candidate LOSE under IRV — and doesn't under STAR |
| [`split_voting/`](split_voting/README_split_voting.md) | the spoiler progression: how choose-one splits, and what each reform does about it |
| [`summability_demo/`](summability_demo/README_summability_demo.md) | two districts + the combined count: STAR subtotals add up, IRV's don't |
| [`paradoxes_and_whoops/`](paradoxes_and_whoops/README_paradoxes_and_whoops.md) | the classics — Tennessee, Condorcet cycles, Ossipoff's centrist, Brams' many-pathologies election |
| [`BV_Library/`](BV_Library/README_BV_Library.md) | real BetterVoting elections across methods (STAR, Approval, Ranked Robin, plurality), imported and verified |
| [`_main/`](_main/README_main.md) | loose comparison files (e.g. STAR-vs-IRV count simplicity) |
| [`divergence_review/`](divergence_review/INDEX.md) | the GENERATED ledger: every curated STAR election re-counted under RCV-IRV / Ranked Robin / Approval, grouped by why they disagree (refreshed by pre-commit) |

The by-method view of every file in the repo is auto-generated at
[`../00_start_here/YAML_test_case_index/`](../00_start_here/YAML_test_case_index/README_YAML_test_case_index.md).
