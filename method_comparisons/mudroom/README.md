# The Mudroom 🥾 — a criterion-failure gallery (deliberately one-sided, openly so)

> **What this is, honestly.** [Arrow](../../00_start_here/topics/arrow_theorem_and_star.md) and [Gibbard–Satterthwaite](../../00_start_here/topics/gibbard_satterthwaite_theorem.md) *prove* that **every** voting method must fail *some* reasonable criterion — there is no perfect method, and there never will be. The Mudroom is the **empirical companion** to those impossibility theorems: a per-method collection of concrete, **runnable** examples of each method on its worst day. The twist is that each reel is written *unfairly on purpose* — curated with zero charity, the way an opponent would — because seeing the mud slung at **every** method (this repo's favorites included) is what makes the [fair pages](../paradoxes_and_whoops/reading_these_fairly.md) trustworthy. The educational core is real; the gleeful *tone* is the indulgence.

**Ground rules for the whole room:**
1. **Every example is real** and documented straight (fair, hedged, rarity-stated) elsewhere in the repo. The curation is the only unfair thing.
2. **Every reel links its own conscience clause** and the grown-up version you'd actually cite in a debate.
3. **No method is spared** — that's the point. If a reel for a method we like looks thin, that's a *to-do*, not a verdict.

*This folder is kept off the main navigation on purpose (see [why](#why-its-tucked-away)). You found it; enjoy responsibly.*

## Why it's tucked away

These pages are **crazy, unfair, and one-sided by design** — they'd misrepresent the state of the art if anyone stumbled on them thinking they were balanced. So they're unlisted from the main navigation and named with a leading underscore: **findable if you go looking, not featured on the front door.** The balanced, cite-in-a-debate versions live where they should — [criteria at a glance](../../00_start_here/topics/criteria_at_a_glance.md), each method's honest-limits page, and [how to read these fairly](../paradoxes_and_whoops/reading_these_fairly.md).

## The actual lesson (if you want one)

Strip the mud off and the Mudroom teaches the single most important fact in voting theory: **there is no perfect method.** Every ballot design and every count trades something away — the honest question isn't "which method is flawless?" (none is) but "which failures can you live with, and how often do they actually bite?" That's the [four-part test](../paradoxes_and_whoops/reading_these_fairly.md) and the whole reason this library exists. The reels just make the impossibility *concrete* — and, by refusing to spare anyone, keep the rest of the repo honest.
